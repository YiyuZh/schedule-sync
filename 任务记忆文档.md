## 当前状态

- 技术栈：FastAPI + SQLAlchemy 2.x + Pydantic v2 + PostgreSQL + Docker Compose
- 当前阶段：云同步服务端第一版可部署，已支持三端同步协议兼容

## 关键决策

- 云端只保存账号、设备、同步快照和变更日志。
- 云端不保存 AI Key，不代理 AI。
- 所有业务数据按 Token 中的 `user_id` 隔离。
- 手机端 `D:\apps\schedule\schedule_mobile` 已进入最终功能基线冻结状态；维护云服务器、部署脚本、Caddy 或管理后台时默认不得修改手机端。
- 云端新增字段、实体扩展或协议修复必须优先保持旧手机端兼容：未知字段可忽略，缺省字段有默认值，旧 sync envelope v1 继续可用。
- 支持 desktop data-wrapped payload 和 mobile flat/envelope payload。
- 支持 `sync_scope=daily_task_status` 的每日任务状态补丁，按 `updated_at` 后改覆盖状态字段，不覆盖任务内容字段。
- 生产环境拒绝默认 JWT、SQLite、示例域名和占位配置。
- 生产 PostgreSQL 用户统一使用 `autsky`；`DATABASE_URL` 写成 `postgresql+psycopg://autsky:<URL编码后的密码>@postgres:5432/schedule_sync`，不能再使用旧用户 `schedule_sync` 或旧邮箱用户。
- 已有 PostgreSQL volume 切换用户时，先运行 `deploy/fix-postgres-user.sh` 创建/更新角色并补齐 schema、表、序列权限。
- 公网网关复用 `/opt/apps/hiremate/Caddyfile` 和 `shared_gateway`。

## 当前能力

- 注册、登录、刷新 Token、退出登录。
- 设备注册和设备管理。
- `/api/sync/push`
- `/api/sync/pull`
- `/api/sync/bootstrap`
- accepted/rejected 明细。
- version conflict reject。
- `daily_task_status` 后改覆盖：合并 `status/completed_at/actual_duration_minutes`，普通编辑仍保留版本冲突保护。
- AI Key 敏感字段清洗。
- CORS 支持 Tauri、Capacitor、Vite、本地调试和正式域名。
- Docker Compose、Alembic、部署脚本、备份脚本、冒烟测试。

## 维护重点

- 改同步协议时必须保持旧 payload 兼容。
- 改 CORS 时同时更新 `.env.example` 和部署文档。
- 改 auth 时必须保留 refresh/logout 和设备登记。
- 不要把 AI Key 写入日志、数据库或响应。
- 上线前必须运行 `deploy/smoke-test.sh`。

## 2026-04-26 每日任务状态准实时同步

- `/api/sync/push` 不新增路由，只兼容 sync envelope v1 可选字段 `sync_scope` 和 `changed_fields`。
- 当 `entity_type=daily_task` 且 `sync_scope=daily_task_status` 时，云端允许旧 `base_version` 状态补丁进入后改覆盖逻辑。
- 新状态按 payload `data.updated_at` 判断是否覆盖云端状态；较旧状态被接受但不覆盖云端，客户端后续 pull 会修正本地。
- 状态补丁只合并 `status`、`completed_at`、`actual_duration_minutes`，保留云端已有 `title/notes/category/task_date` 等内容字段。
- 测试已覆盖较新状态覆盖、较旧状态不覆盖、标题备注不被状态补丁改写。

## 2026-04-26 管理员可视化后台

- 新增后台入口：`https://schedule-sync.zenithy.art/admin`。
- 管理员邮箱固定通过环境变量 `ADMIN_EMAIL` 配置，当前约定为 `autsky6666@gmail.com`。
- 生产环境必须配置 `ADMIN_PASSWORD_HASH`，禁止使用明文 `ADMIN_PASSWORD`；明文只允许本地开发临时调试。
- 新增后台 API：`/api/admin/login`、`/api/admin/me`、`/api/admin/overview`、`/api/admin/users`、`/api/admin/users/{user_id}`、`POST /api/admin/users/{user_id}/password`、`DELETE /api/admin/users/{user_id}`。
- 后台页面使用 FastAPI 托管原生静态文件：`app/admin_static/index.html`、`admin.css`、`admin.js`，不引入 Node 构建链。
- 可视化指标包含：总用户数、近 30 日活跃用户、设备数、同步记录数、今日活跃、近 30 日每日活跃图。
- 用户列表支持搜索、分页、详情查看、重置密码和删除。重置密码会写入密码哈希，并可让该用户现有 Refresh Token 失效；删除用户会清理设备、Refresh Token、用户设置、同步记录、同步变更；禁止删除管理员邮箱对应用户。
- Caddy 不需要新增站点，继续由 `schedule-sync.zenithy.art` 反代 `schedule-sync-api:8000`。
- 验证命令已扩展到 `python -m pytest -q`，当前后台相关测试包含管理员登录、权限隔离、统计、搜索、重置密码、删除清理和管理员账号删除保护。
