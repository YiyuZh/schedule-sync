## 当前状态

- 项目路径：`D:\apps\schedule_sync_server`
- 服务器部署路径：`/opt/apps/schedule-sync`
- 正式同步域名：`https://schedule-sync.zenithy.art`
- 公网 Caddy：`/opt/apps/hiremate/Caddyfile`，容器 `hiremate-caddy`，网络 `shared_gateway`
- 技术栈：FastAPI + SQLAlchemy 2.x + Pydantic v2 + PostgreSQL + Docker Compose
- 当前阶段：云同步服务端第一版可部署，已支持三端同步协议兼容

## 关键决策

- 云端只保存账号、设备、同步快照和变更日志。
- 云端不保存 AI Key，不代理 AI。
- 所有业务数据按 Token 中的 `user_id` 隔离。
- 支持 desktop data-wrapped payload 和 mobile flat/envelope payload。
- 生产环境拒绝默认 JWT、SQLite、示例域名和占位配置。
- 公网网关复用 `/opt/apps/hiremate/Caddyfile` 和 `shared_gateway`。

## 当前能力

- 注册、登录、刷新 Token、退出登录。
- 设备注册和设备管理。
- `/api/sync/push`
- `/api/sync/pull`
- `/api/sync/bootstrap`
- accepted/rejected 明细。
- version conflict reject。
- AI Key 敏感字段清洗。
- CORS 支持 Tauri、Capacitor、Vite、本地调试和正式域名。
- Docker Compose、Alembic、部署脚本、备份脚本、冒烟测试。

## 维护重点

- 改同步协议时必须保持旧 payload 兼容。
- 改 CORS 时同时更新 `.env.example` 和部署文档。
- 改 auth 时必须保留 refresh/logout 和设备登记。
- 不要把 AI Key 写入日志、数据库或响应。
- 上线前必须运行 `deploy/smoke-test.sh`。
